## 目标

本阶段交付一个可用的素材库 MVP：用户可以创建或打开素材库，默认把导入文件复制进素材库目录，数据库和管理文件只放在 `.asset-manager` 下，能够按文件名、标签和备注搜索，并能预览图片、视频、音频和文本片段。

本次实现已按用户确认调整为：导入素材时依据软件界面当前所处文件夹决定复制位置，不创建固定 `assets` 文件夹。

## 实现原则

- 素材库默认采用 copy-on-import，不只记录原路径。
- 导入目标由 UI 当前选中的素材库文件夹决定。
- 不创建 `assets` 目录；用户创建的目录或导入文件夹本身属于素材内容，不属于管理文件污染。
- 素材库根目录唯一自动生成的管理目录是 `.asset-manager`。
- 数据库、缩略图缓存、临时文件、日志和库内状态都必须写入 `.asset-manager`。
- 不在本阶段引入插件市场、Shell Extension、云同步或多端协作。
- 保持与当前 Windows 文件交互 POC 的拖入、拖出、剪贴板通路一致。

## 回滚方案

如果导入或数据库结构出现问题，优先回退素材库导入逻辑，保留现有 Windows 文件交互 POC。`.asset-manager` 内的数据可通过版本化迁移恢复；若迁移失败，可删除素材库内的 `asset-manager.db` 后重新初始化，不影响库内素材文件本身。

## 备注

- 这一阶段不做插件系统扩展，不做 Shell Extension，不做在线插件市场。
- 音频和视频预览先做可用版本，不追求编辑级能力。
- 首批目标仍是 Windows 11，QQ 和资源管理器验证沿用现有 POC 路径。
- QQ 客户端真实拖放仍需要在本机客户端中做人工验收；本次代码保留 `WindowsFileTransferService` 的文件拖出和剪贴板 FileDrop 通路。
