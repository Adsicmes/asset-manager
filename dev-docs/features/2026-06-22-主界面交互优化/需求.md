# 主界面交互优化

> 创建日期: 2026-06-22 16:12
> 最后更新: 2026-06-22 16:12
> 作者: Adsicmes
> 状态: 已确认

## 背景

当前素材管理器主界面顶部工具栏使用 WrapPanel 平铺了 14 个控件（2 个下拉框 + 8 个按钮 + 1 个复选框 + 标签文本），涵盖全局设置、素材库管理、文件操作、同步/复制等不同层级的功能。所有操作没有视觉分组，用户需要逐个扫描才能找到目标按钮。

具体问题：

1. 高频操作（导入、新建文件夹、搜索）与低频操作（注册素材库、语言切换）混在一起，无法快速定位。
2. "导入文件"和"导入文件夹"是同一意图的两个入口，占用两个按钮位。
3. 语言切换是极低频操作，不应占据主工具栏位置。
4. 右侧详情面板在未选择素材时显示空白，浪费 360px 水平空间。
5. 搜索栏位置不够突出，夹在路径文字和按钮之间。

## 目标

重新组织主界面布局，将低频操作收入菜单栏，工具栏只保留高频操作，详情面板改为按需展开，使整体交互更清晰高效。

## 用户故事

作为素材使用者，我希望打开应用后一眼就能看到我最常用的操作（导入、搜索、同步），而不是在十几个按钮里找目标。

作为素材使用者，我希望不常用的操作（注册素材库、语言切换）不占据视觉焦点，但在需要时仍然可达。

作为素材使用者，我希望浏览素材时有更大的内容区域，只在选中素材后才显示详情面板。

## 功能描述

### 核心功能

- [ ] 引入经典菜单栏（Menu），收纳低频操作
- [ ] 精简工具栏，只保留高频操作按钮
- [ ] 合并"导入文件"和"导入文件夹"为一个带下拉的 SplitButton
- [ ] 语言切换从工具栏移除，移入菜单"设置"或独立设置对话框
- [ ] 右侧详情面板改为选中素材后才展开，未选中时隐藏
- [ ] 搜索栏位置提升，固定在工具栏右侧

### 菜单栏结构

| 菜单 | 包含项 |
|------|--------|
| 文件 | 注册素材库、新建文件夹、导入文件、导入文件夹、新建文本 |
| 编辑 | 复制 |
| 视图 | 卡片字段设置、详情面板显隐切换 |
| 素材库 | 刷新素材库、同步、低干扰导入开关 |
| 设置 | 语言选择 |

### 精简后的工具栏

```text
[素材库▾ MediaLibrary]  |  [新建文件夹] [导入▾] [同步]  |  [🔍 搜索...]
```

工具栏从 14 个控件减少到约 5 个（素材库选择器、新建文件夹、导入 SplitButton、同步、搜索框）。

### 详情面板行为变更

- 默认隐藏右侧详情面板（DetailsColumn Width=0, DetailsSplitterColumn Width=0）
- 选中素材后，详情面板展开（DetailsColumn Width=360）
- 可通过菜单"视图 > 显示详情面板"强制常驻显示
- 取消选中时自动收起（除非用户选择了常驻模式）

### 边界条件

- 菜单栏中的操作与原有按钮行为完全一致，不改变业务逻辑
- 工具栏按钮的可用状态（CanInteract）逻辑不变
- 详情面板展开/收起应有平滑过渡或至少无闪烁
- 键盘快捷键：导入（Ctrl+I）、新建文件夹（Ctrl+Shift+N）、搜索（Ctrl+F）、同步（F5）
- SplitButton 的下拉菜单包含"导入文件"和"导入文件夹"两个选项，主按钮点击执行"导入文件"

## 非功能需求

- 性能: 菜单和面板展开/收起不应引入可感知的延迟
- 兼容性: 不引入额外 NuGet 包，使用 WPF 内置控件实现（Menu、MenuItem、ContextMenu）
- 可维护性: SplitButton 如无合适内置控件，可用 Button + ContextMenu 组合实现

## 验收标准

1. 打开应用后，工具栏可见按钮不超过 5 个（不含搜索框）
2. 菜单栏包含"文件/编辑/视图/素材库/设置"五个顶级菜单
3. 所有原有功能通过菜单或工具栏均可触达，无功能丢失
4. 语言切换不出现在主工具栏中
5. 未选中素材时，右侧详情面板不占据空间
6. 选中素材后，详情面板自动展开并显示素材信息
7. "导入"按钮点击后可选择导入文件或导入文件夹
8. 搜索框在工具栏右侧固定可见
9. Ctrl+I / Ctrl+Shift+N / Ctrl+F / F5 快捷键可用

## 优先级与排期

- 优先级: P1
- 预计工期: 分 3 个阶段

### 阶段划分

1. Phase 1: 引入菜单栏 + 精简工具栏（核心改动）
2. Phase 2: 详情面板按需展开
3. Phase 3: 快捷键绑定 + 细节打磨
