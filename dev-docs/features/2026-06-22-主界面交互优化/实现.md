## 目标

将主窗口从"14 控件平铺工具栏 + 常驻详情面板"改造为"菜单栏 + 精简工具栏 + 按需详情面板"布局。不改变任何业务逻辑，仅重组 UI 入口。

## 实现步骤

### Phase 1: 引入菜单栏 + 精简工具栏

核心改动，减少工具栏按钮数量，将低频操作收入菜单。

- [x] 1.1 在 MainWindow.xaml 的 DockPanel 顶部添加 `<Menu>` 控件
- [x] 1.2 定义菜单结构：文件、编辑、视图、素材库、设置
- [x] 1.3 为每个 MenuItem 绑定已有的 Click 事件处理器
- [x] 1.4 从 WrapPanel 移除以下按钮/控件：
  - 语言标签 + LanguageBox（移入"设置"菜单）
  - 注册素材库按钮（移入"文件"菜单）
  - 刷新素材库按钮（移入"素材库"菜单）
  - 导入文件按钮（移入"文件"菜单 + 工具栏 SplitButton）
  - 导入文件夹按钮（移入"文件"菜单 + 工具栏 SplitButton 下拉）
  - 低干扰导入 CheckBox（移入"素材库"菜单，使用 IsCheckable MenuItem）
  - 新建文本按钮（移入"文件"菜单）
  - 复制按钮（移入"编辑"菜单）
- [x] 1.5 工具栏保留：素材库选择下拉、新建文件夹按钮、导入 SplitButton、同步按钮
- [x] 1.6 实现导入 SplitButton（Button + ContextMenu 组合）
  - 主按钮点击 → 执行导入文件（ImportFiles_Click）
  - 下拉箭头点击 → 弹出 ContextMenu（导入文件 / 导入文件夹）
- [x] 1.7 搜索栏移至工具栏右侧，固定可见
- [x] 1.8 添加国际化资源键：MenuFile、MenuEdit、MenuView、MenuLibrary、MenuSettings 等
- [x] 1.9 在 Strings.zh-CN.xaml 和 Strings.en-US.xaml 中添加菜单项文本

### Phase 2: 详情面板按需展开

- [x] 2.1 修改默认状态：`_isDetailsPanelExpanded` 初始值改为 `false`
- [x] 2.2 在 AssetList 选中事件中，当选中首个素材时自动展开详情面板
- [x] 2.3 在取消全部选中时自动收起详情面板（除非用户通过菜单设为常驻）
- [x] 2.4 添加 `_isDetailsPanelPinned` 字段，控制常驻模式
- [x] 2.5 在"视图"菜单添加"固定详情面板"选项（IsCheckable），控制常驻
- [x] 2.6 持久化 pinned 状态到 UiSettingsStore
- [x] 2.7 面板展开/收起时设置 DetailsColumn.Width 和 DetailsSplitterColumn.Width

### Phase 3: 快捷键 + 细节

- [x] 3.1 注册快捷键：
  - Ctrl+I → 导入文件
  - Ctrl+Shift+N → 新建文件夹
  - Ctrl+F → 聚焦搜索框
  - F5 → 同步
- [x] 3.2 在 MenuItem 上显示快捷键提示（InputGestureText）
- [x] 3.3 工具栏按钮添加 ToolTip，包含快捷键说明
- [x] 3.4 移除顶部 Border 中已清空的 WrapPanel（如果为空）
- [x] 3.5 调整工具栏 Margin/Padding 确保视觉平衡

## 回滚方案

所有改动仅限 Desktop 项目 UI 层，不涉及 Domain / Application / Infrastructure。如果出现问题，git revert 对应提交即可恢复原始工具栏布局。

## 备注

- SplitButton 在 WPF 内置控件中不存在，采用 Button + 右侧小箭头 Button + ContextMenu 的经典组合实现，不引入第三方库。
- 菜单栏中"设置 > 语言"子菜单使用 MenuItem + 子 MenuItem 列表实现语言选择，绑定现有 LanguageOptions 数据源。
- 详情面板展开/收起复用已有的 `ApplyDetailsPanelState` 方法。
