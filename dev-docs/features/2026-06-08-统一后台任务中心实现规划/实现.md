## 目标

本轮实现要交付一套统一的后台任务中心，让导入、同步、缩略图生成和后续插件任务不再各自维护零散状态，而是通过同一套任务模型向 UI 暴露状态、进度、结果和可取消能力。

第一版范围只覆盖单机桌面内任务编排，不做分布式调度、不做跨进程持久化恢复、不做复杂优先级抢占。

## 实现原则

- 后台任务中心不是 Domain 概念，不能放进 `AssetManager.Domain`。
- Application 层定义稳定的任务状态模型和上报端口，不能直接依赖 WPF `Dispatcher`、`ObservableCollection` 或控件类型。
- Desktop 层负责把任务快照适配成状态栏、任务列表和通知，不反向污染 Application。
- 长任务本身仍归属于各自用例或基础设施服务，任务中心只负责注册、状态聚合、取消和结果发布，不接管业务规则。
- 第一步优先收口已有后台任务：导入、同步、缩略图；插件任务接入点先预留，不在本轮做完整插件执行器。
- 状态栏只显示当前最重要的任务摘要；完整列表、历史和诊断进入独立任务面板。

## 实现步骤

### Phase 1: Application 层任务模型与端口

- [x] 在 `src/AssetManager.Application/BackgroundTasks/` 新增统一任务模型，例如 `BackgroundTaskId`、`BackgroundTaskKind`、`BackgroundTaskState`、`BackgroundTaskProgress`、`BackgroundTaskSnapshot`。
- [x] 定义 `IBackgroundTaskCenter` 或等价输入端口，提供 `Start`、`ReportProgress`、`Complete`、`Fail`、`CancelRequested` 查询等能力。
- [x] 定义面向 UI/适配层读取的只读查询端口，例如 `IBackgroundTaskFeed`，用于获取当前任务快照流或最新快照集合。
- [x] 定义简单 DTO，跨层只传字符串、枚举、数字和时间戳，不传 WPF 或具体线程对象。
- [x] 明确任务结果分类：成功、失败、取消、部分成功。

### Phase 2: Desktop 适配器与统一状态源

- [x] 在 `src/AssetManager.Desktop/` 新增后台任务中心适配器，实现 Application 端口并负责 UI 线程派发。
- [x] 抽出当前状态栏逻辑，改为绑定统一任务摘要，而不是直接拼装导入/缩略图专用字段。
- [x] 新增 `BackgroundTaskRow` 或等价 ViewModel，承载任务标题、阶段、进度、状态、开始时间、结束时间、错误摘要和取消按钮可见性。
- [x] 新增任务面板入口，至少支持查看当前运行任务和最近完成任务。
- [x] 定义任务摘要优先级规则：导入 > 同步 > 插件任务 > 缩略图批处理 > 空闲。

### Phase 3: 接入现有导入、同步和缩略图任务

- [x] 导入流程接入统一任务中心，至少上报开始、目标文件夹、源项数量、完成和失败状态。
- [x] `FileSystemAssetContentStore` 补充可选进度回调或任务 reporter，支持按文件/字节上报复制进度。
- [x] 同步流程接入统一任务中心，上报扫描中、比对中、写回中和最终统计。
- [x] `AssetThumbnailLoadCoordinator` 改为通过统一任务中心注册“缩略图批处理任务”，不再自己维护独立状态事件。
- [x] 保证这些任务在没有 UI 的测试环境下也能运行，任务中心实现需允许 no-op 或内存实现。

### Phase 4: 取消、失败诊断与任务历史

- [x] 为导入、同步、缩略图任务统一接入 `CancellationToken` 映射，任务中心可请求取消。
- [x] 失败任务统一记录错误摘要、异常类型和发生阶段；状态栏只显示短消息，详情面板显示完整信息。
- [x] 增加有限历史缓存，例如最近 50 条任务，避免无限增长。
- [x] 定义任务去重/合并规则，避免每一张缩略图都生成一个独立可见任务。
- [x] 为未来插件任务预留 `TaskOwner = core/plugin`、`PluginId` 等字段，但本轮不做插件执行细节。

### Phase 5: 验证与架构收口

- [x] 添加 Application 层单元测试，覆盖任务状态流转、取消和失败语义。
- [x] 添加 Desktop 层测试或最小可测试适配逻辑，覆盖任务摘要优先级和列表更新。
- [ ] 手动验证并记录：导入大文件夹时状态栏和任务面板能持续更新，不出现卡死或假完成。
- [ ] 手动验证：缩略图批处理运行时，状态栏摘要会切换为缩略图任务，结束后恢复空闲。
- [ ] 手动验证：同步失败或导入失败时，状态栏显示失败摘要，任务面板可查看详情。

## 回滚方案

如果统一任务中心接入后导致现有导入、同步或缩略图行为不稳定，优先保留 Application 层任务模型和 no-op 实现，回退 Desktop 层绑定和具体任务接入点，恢复当前分散但可运行的状态管理方式。这样不会影响素材库数据结构和导入结果，只回退任务状态展示与编排。

## 备注

- 当前项目已经存在三个后台任务雏形：导入中的 `_isImporting` 状态、同步操作的执行窗口、缩略图加载协调器 `AssetThumbnailLoadCoordinator`。统一任务中心应优先收口这三处，而不是从零引入一个完全平行的新系统。
- 第一版任务中心建议采用“内存态 + UI 绑定”的轻量实现，不把它设计成数据库持久化队列。否则会过早把简单问题做重。
- 任务中心的接口归属应遵守依赖反转：Application 定义端口，Desktop/Infrastructure 提供实现。不要让 `LibraryApplicationService` 直接知道状态栏、`Dispatcher` 或 `ObservableCollection`。
- 插件任务后续应视为任务中心的普通生产者，而不是单独再造一套插件任务状态系统。
