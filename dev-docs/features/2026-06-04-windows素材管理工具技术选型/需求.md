# Windows 素材管理工具技术选型需求

> 创建日期: 2026-06-04 10:23
> 最后更新: 2026-06-04 10:23
> 作者: Adsicmes
> 状态: 草稿

## 背景

项目计划开发一个仅面向 Windows 的素材管理工具。该工具需要与操作系统进行强文件交互，包括和资源管理器之间复制、粘贴、拖入、拖出，以及将素材拖动或发送到 QQ 等社交软件。系统还需要具备强扩展性，允许后续通过插件扩展素材导入、元数据提取、预览、批处理、外部应用投递等能力。

## 目标

选择一套适合 Windows 桌面、Shell/OLE 文件交互和插件扩展的技术栈。选型应优先保证文件互操作可靠性、插件边界清晰、后续可维护，而不是追求跨平台。

## 用户故事

作为素材管理工具用户，我希望可以像使用资源管理器一样复制、粘贴、拖动素材，以便快速把素材放入项目目录、桌面、QQ 聊天窗口或其他软件。

作为插件开发者，我希望通过稳定的插件 SDK 扩展素材处理能力，以便在不修改主程序的情况下增加格式识别、预览、导出和自动化能力。

## 功能描述

### 核心功能

- [ ] 素材入库、分类、标签、搜索、预览和文件定位。
- [ ] 支持资源管理器拖入、拖出，支持复制/剪切/粘贴文件语义。
- [ ] 支持向 QQ 等外部应用投递真实文件路径或临时物化文件。
- [ ] 支持插件扩展元数据提取、预览、缩略图、导入导出、批处理命令和 UI 面板。
- [ ] 支持本地索引、缩略图缓存、后台扫描和文件变更同步。

### 边界条件

- 素材不是本地真实文件时，需要先物化到临时目录，再通过标准文件拖放或剪贴板格式交给外部应用。
- 外部应用只接受文件路径时，不能只传自定义对象或内存流。
- 大文件复制、跨盘移动、网络路径和权限受限目录需要使用 Shell 语义处理进度、冲突和失败。
- 第三方插件不能默认获得完整文件系统访问权限，需要能力声明、用户授权和隔离运行策略。

## 非功能需求

- 性能: 大量素材索引、搜索、缩略图生成应后台化，不阻塞 UI。
- 兼容性: 主目标为 Windows 11；如必须覆盖 Windows 10，需要单独确认支持范围，因为 Windows 10 常规支持已于 2025-10-14 结束。
- 安全性: 插件包需要清单、签名或哈希校验；不可信插件应运行在独立进程。
- 可维护性: 主程序、文件集成、插件宿主、索引服务、UI 应分层，避免插件直接耦合内部实现。

## 验收标准

1. 技术栈能覆盖 Windows Shell 文件复制、剪贴板、拖放、资源管理器互操作和 QQ 文件投递场景。
2. 插件系统能同时支持可信 .NET 插件和不可信进程外插件。
3. 本地数据库和搜索方案能满足资产元数据检索与全文检索。
4. 打包方案不会妨碍插件安装、缓存写入和未来 Shell 集成。

## 优先级与排期

- 优先级: P0
- 预计工期: 待实现规划确认
- 依赖项: Windows Shell/OLE 交互原型、插件宿主原型、索引与缩略图原型
