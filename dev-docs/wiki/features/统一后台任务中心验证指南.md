# 统一后台任务中心验证指南

> 创建日期: 2026-06-08 18:47
> 最后更新: 2026-06-08 18:47
> 作者: Adsicmes

## 概述

本文档记录如何在本地 Windows 开发环境中验证统一后台任务中心。目标是用最少的手工准备，覆盖导入任务、缩略图批处理、同步任务、失败任务和语言/素材库切换后的状态一致性。

## 验证前准备

先在仓库根目录执行准备脚本：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev\Prepare-BackgroundTaskValidation.ps1
```

脚本会输出以下路径：

- `LibraryRoot`：用于注册为素材库的位置。
- `ImportSource`：用于导入的大文件夹，包含图片、文本和音频文件。
- `SuggestedImportFailurePath`：用于锁文件制造导入失败的源文件。

同时会在 `artifacts/manual-validation/background-task-center/<timestamp>/` 下创建验证工作区。

## 在 Rider 中启动应用

1. 在 Rider 打开 `AssetManager.sln`。
2. 把运行目标设为 `src/AssetManager.Desktop/AssetManager.Desktop.csproj`。
3. 启动应用，确认主窗口能正常打开。
4. 点击 `Register Library`，选择准备脚本输出的 `LibraryRoot`。

## 验证导入任务与缩略图聚合任务

建议先打开主窗口顶部的 `Low-impact Import`，这样进度更容易观察。

步骤：

1. 点击底部状态栏中的 `View Tasks` 打开任务窗口。
2. 点击 `Import Folder`，选择准备脚本输出的 `ImportSource`。
3. 观察状态栏和任务窗口。

预期结果：

- 状态栏先显示导入任务摘要，而不是缩略图任务。
- 任务窗口里只出现一个导入任务，进度按文件数持续增长。
- 导入完成后，会出现单个缩略图批处理任务，而不是每张图片一个任务。
- 缩略图任务完成后，状态栏恢复为空闲或最近任务摘要。

## 验证同步任务阶段与统计

先完成一次导入，然后在另一个 PowerShell 窗口运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev\Mutate-BackgroundTaskValidationLibrary.ps1 -LibraryRoot "<LibraryRoot>"
```

然后回到应用，点击 `Sync`。

预期结果：

- 状态栏依次出现扫描、核对、登记新增素材等阶段性文案。
- 任务窗口中的同步任务是单个聚合任务。
- 同步完成后，状态栏和主状态消息中会显示更新、移动、丢失和新增统计。

## 验证同步失败任务

先找到库内数据库：

```text
<LibraryRoot>\.asset-manager\asset-manager.db
```

在另一个 PowerShell 窗口运行锁文件脚本：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev\Hold-ExclusiveLock.ps1 -Path "<LibraryRoot>\.asset-manager\asset-manager.db" -Seconds 60
```

在锁持有期间回到应用，点击 `Sync`。

预期结果：

- 同步任务进入失败状态。
- 状态栏显示短失败摘要。
- 任务窗口的错误列能看到异常类型和错误消息。

## 验证导入失败任务

使用准备脚本输出的 `SuggestedImportFailurePath`，在另一个 PowerShell 窗口运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev\Hold-ExclusiveLock.ps1 -Path "<SuggestedImportFailurePath>" -Seconds 60
```

锁持有期间，在应用中点击 `Import Files`，只选择这一个被锁定的文件。

预期结果：

- 导入任务进入失败状态。
- 状态栏显示带阶段信息的失败摘要。
- 任务窗口错误列能看到异常类型和错误消息。

## 验证语言切换、素材库切换与关闭窗口

语言切换：

1. 在任务窗口保持打开的情况下切换 `zh-CN` / `en-US`。
2. 确认后续新任务文案使用新语言。
3. 确认当前窗口没有异常或崩溃。

素材库切换：

1. 再注册一个空素材库目录。
2. 在两个素材库之间切换。
3. 确认任务面板和状态栏不会残留上一个素材库的运行中任务。

关闭窗口：

1. 在缩略图任务运行时直接关闭主窗口。
2. 重新启动应用。
3. 确认不会出现悬挂任务状态或启动崩溃。
